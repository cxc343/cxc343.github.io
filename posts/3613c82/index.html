一、Git 常用命令说明

1. 基础命令

命令说明
git status查看当前分支状态，哪些文件被修改、哪些文件暂存
git add <file>将文件加入暂存区
git commit -m "msg"提交暂存区到本地分支
git log查看提交历史
git diff查看工作区修改内容
git branch查看本地分支列表
git branch -a查看本地 + 远程分支
git checkout <branch>老版切换分支（新版推荐 git switch）
git switch <branch>切换分支
git switch -c <branch>新建并切换分支
git switch -切回上一个分支
git merge <branch>合并指定分支到当前分支
git rebase <branch>将当前分支变基到指定分支
git fetch <remote>获取远程仓库最新信息（不自动合并）
git pull拉取远程更新并合并到当前分支
git push推送本地分支到远程
git remote -v查看远程仓库信息
git tag <name>创建标签
git restore <file>恢复文件修改（新版替代部分 checkout 用法）
git restore --staged <file>恢复暂存区文件修改
git worktree add <dir> <branch>为分支创建独立工作目录
git worktree list查看已有 worktree
git worktree remove <dir>删除 worktree 目录

二、实际场景及命令组合

场景1：从远程仓库拉分支到本地

说明：

  • 本地创建 feature/login 分支，并追踪远程分支
  • 新版 switch 命令比 checkout 更清晰

场景2：基于本地分支新建分支

或者一步到位：

场景3：多层 Feature 分支开发

如果上游更新：

  • --update-refs 会自动更新子 feature 分支
  • 避免分支断裂

场景4：同时开发多个 Feature（Worktree）

目录结构：

  • 每个工作树独立目录，互不影响
  • IDE 可以同时打开多个目录

场景5：线上 Hotfix

说明：

  • Hotfix 与当前开发分支隔离
  • 不用 stash 切换回 master

场景6：代码 Review

  • Review 完即可删除目录
  • 原开发分支不受影响

场景7：版本对比 / 升级

  • 保留当前分支开发状态
  • 升级或改动在独立目录里完成
  • IDE 同时打开两套代码进行对比

✅ 总结：

  1. 新版 switch + restore 命令代替部分 checkout，更清晰。

  2. worktree 可让一个仓库同时存在多个独立目录，适合：

    • 多 Feature 并行
    • 热修复
    • 代码 Review
    • 大版本升级对比

  3. rebase --update-refs 可自动维护多层 Feature 分支关系。

  4. 常用命令组合起来可以大幅提升日常开发效率，减少切分支、stash 的负担。
